# SourceOS Local Agent Runtime Specification

Status: draft
Owner: SourceOS / SocioProphet platform
Scope: Nix-governed local agents, launchd/systemd persistence, Podman-backed agents, local-runtime diagnostics

## 1. Purpose

SourceOS local agents must be reproducible, explainable, observable, reversible, and bounded. Nix gives us deterministic artifacts; the SourceOS local-agent runtime gives those artifacts a safe operational contract.

This specification exists because a local `node-commander` service landed as a root-owned, system-wide macOS LaunchAgent in `/Library/LaunchAgents`, invoked a Nix store wrapper, ran Podman directly, depended on noninteractive Google registry authentication, wrote logs to `/tmp`, and used unbounded launchd restart behavior. The result was a respawn loop that looked like suspicious persistence on an unmanaged Mac.

This class of failure is not acceptable for SourceOS. Every persistent local agent must have a declared owner, scope, runtime, auth model, logging model, health model, uninstall path, and quarantine path.

## 2. Design principles

1. Reproducible does not mean opaque. Nix store paths may be implementation details, but the operator-facing control surface must be stable and legible.
2. Persistence is a privileged product surface. Installing launchd/systemd persistence requires preflight, status, logs, and rollback.
3. Local agents fail closed. Missing Podman machines, missing images, broken sockets, stale auth, or invalid credentials must emit one clear diagnostic and avoid restart storms.
4. User-session agents run as user agents. System daemons are only allowed when system scope is explicitly required and justified.
5. No ambient interactive auth in noninteractive services. A launchd/systemd service must not depend on a human shell session, `gcloud auth login`, or Docker credential helper refresh.
6. Logs are product artifacts. Product services must not write primary logs to `/tmp`.
7. Quarantine is first-class. Every agent install must be reversible with audit capture.
8. Linux is the primary target. macOS launchd support exists for development hosts and cross-device capture, but the Linux systemd contract is canonical.

## 3. Required local-agent declaration

Every local agent MUST declare the following fields:

```nix
sourceos.localAgents.<name> = {
  enable = true;
  scope = "user"; # user | system
  runtime = "podman"; # podman | native | nix-app | systemd
  command = null; # required for native/nix-app runtimes
  image = "localhost/sourceos/<agent>:<tag>"; # required for podman runtime
  containerName = "<agent>";

  owner = {
    project = "SourceOS";
    component = "local-runtime";
    contact = "platform";
  };

  auth = {
    mode = "empty-authfile"; # empty-authfile | service-account | none | forbidden-ambient
    authfile = "$HOME/.config/containers/sourceos-empty-auth.json";
    allowAmbientDockerConfig = false;
    allowInteractiveCredentialHelper = false;
  };

  podman = {
    connection = "podman-machine-default";
    pull = "never"; # never | explicit | always
    requireMachine = true;
    requireLocalImage = true;
    rootless = true;
  };

  lifecycle = {
    runAtLoad = true;
    keepAlive = false;
    restart = "manual"; # manual | bounded | supervised
    maxRestartsPerHour = 3;
    startupTimeoutSeconds = 20;
    shutdownTimeoutSeconds = 10;
  };

  logs = {
    app = "$HOME/Library/Logs/SourceOS/<agent>.log";
    stdout = "$HOME/Library/Logs/SourceOS/<agent>.launchd.out.log";
    stderr = "$HOME/Library/Logs/SourceOS/<agent>.launchd.err.log";
  };

  health = {
    type = "podman-container";
    expected = "running";
    command = "podman --connection podman-machine-default ps --filter name=<agent>";
  };

  quarantine = {
    captureServiceDefinition = true;
    captureLogs = true;
    captureRuntimeState = true;
    captureImageMetadata = true;
    captureRedactedAuthConfig = true;
  };
};
```

## 4. Persistence rules

### 4.1 macOS launchd

User-session agents MUST be installed under:

```text
~/Library/LaunchAgents/<label>.plist
```

System daemons MUST be installed under:

```text
/Library/LaunchDaemons/<label>.plist
```

The following are prohibited unless explicitly waived in the declaration and accepted by policy checks:

- Root-owned user-session agents under `/Library/LaunchAgents`.
- `KeepAlive=true` without bounded restart policy.
- Primary logs under `/tmp`.
- Direct `/bin/sh -c` launch commands not generated by SourceOS tooling.
- Direct runtime reference to remote registry images for local/offline agents.
- Interactive credential helpers in launchd services.
- LaunchAgent installation before preflight passes.

Launchd adapters MUST detect:

- Disabled overrides.
- Duplicate labels.
- Stale services.
- Invalid plist syntax.
- Invalid ownership or permissions.
- Missing wrapper targets.
- Bootstrap input/output errors.
- Excessive run count.

### 4.2 Linux systemd

User agents SHOULD be generated as systemd user units:

```text
~/.config/systemd/user/sourceos-agent@<name>.service
```

System agents SHOULD be generated as:

```text
/etc/systemd/system/sourceos-agent@<name>.service
```

Systemd services MUST use bounded restart behavior. `Restart=always` is prohibited by default. If restart is enabled, the unit MUST include `StartLimitBurst` and `StartLimitIntervalSec`.

Podman Quadlet MAY be used when it improves clarity, but the SourceOS local-agent declaration remains canonical.

## 5. Preflight contract

No persistent agent may be installed or enabled until preflight passes, unless the operation is explicitly `stage-only`.

Required preflight checks:

1. Host platform detected.
2. Backend adapter available: launchd or systemd.
3. Runtime available: Podman/native/Nix app.
4. Podman binary exists when runtime is Podman.
5. Podman version supported.
6. Podman machine exists when required.
7. Podman machine is running when required.
8. Podman socket reachable.
9. Local image exists when `pull = "never"`.
10. Image uses allowed runtime tag.
11. Authfile exists and matches declared auth mode.
12. Host Docker/containers auth config cannot poison runtime.
13. Container name is free or safely replaceable.
14. Log directory exists and is writable.
15. Service label is not disabled.
16. Service label is not already owned by a conflicting service.
17. Generated plist/unit lints successfully.
18. Generated wrapper lints successfully.

Preflight output MUST be machine-readable and human-readable.

Example:

```text
node-commander preflight: failed
- podman machine: exists but not running
- socket: 127.0.0.1 refused connection
- image: local image present
- auth: empty-authfile present
- action: run `sourceos-agent machine start podman-machine-default`
- persistence: staged but not enabled
```

## 6. Runtime auth policy

Local runtime MUST NOT depend on ambient Docker or Google Cloud credential helpers.

Allowed modes:

### 6.1 empty-authfile

For already-local images:

```text
--authfile $HOME/.config/containers/sourceos-empty-auth.json
```

The file content MUST be:

```json
{"auths":{}}
```

Runtime image names SHOULD use `localhost/...` tags.

### 6.2 service-account

For noninteractive remote pull/update flows, the agent must use a service-grade credential mechanism. Interactive `gcloud auth login` is not valid service auth.

### 6.3 explicit interactive pull

Developers may run an explicit image pull command interactively. Persistent runtime still uses local tags and noninteractive-safe auth.

## 7. Image provenance

Every Podman-backed agent MUST record provenance:

```json
{
  "agent": "node-commander",
  "runtimeImage": "localhost/socioprophet/node-commander:dev-boot",
  "sourceImage": "us-central1-docker.pkg.dev/socioprophet-web/node-commander/node-commander:dev-boot",
  "imageId": "sha256:...",
  "pulledAt": "...",
  "runtimeAuth": "empty-authfile",
  "pullAuth": "interactive-or-service-account"
}
```

Runtime services MUST prefer `runtimeImage`, not `sourceImage`.

## 8. Operator commands

The SourceOS CLI MUST expose:

```text
sourceos-agent install <name>
sourceos-agent stage <name>
sourceos-agent start <name>
sourceos-agent stop <name>
sourceos-agent restart <name>
sourceos-agent status <name>
sourceos-agent logs <name>
sourceos-agent doctor <name>
sourceos-agent preflight <name>
sourceos-agent quarantine <name>
sourceos-agent uninstall <name>
```

`sourceos doctor local-runtime` MUST inspect all local agents and report:

- MDM/profile state on macOS development hosts.
- launchd/systemd state.
- Nix generation/source.
- Podman machine state.
- Container state.
- Image provenance.
- Auth mode.
- Log paths.
- Last exit reason.
- Restart/run count.
- Suspicious persistence patterns.

## 9. Quarantine contract

`sourceos-agent quarantine <name>` MUST:

1. Stop the service.
2. Disable the service.
3. Move service definition to a timestamped quarantine directory.
4. Copy logs.
5. Capture launchd/systemd status.
6. Capture Podman machine info.
7. Capture container inspect output when present.
8. Capture image inspect output when present.
9. Capture redacted auth configuration.
10. Emit `manifest.json`.
11. Emit `remediation.md`.

Example quarantine directory:

```text
~/Desktop/sourceos-quarantine/node-commander-20260504-184500/
  manifest.json
  service.plist.disabled
  node-commander.log
  node-commander.err.log
  podman-info.json
  image-inspect.json
  container-inspect.json
  auth-redacted.json
  remediation.md
```

## 10. Nix activation safety

Nix activation MUST follow this order:

1. Evaluate local-agent declaration.
2. Generate wrapper.
3. Lint wrapper.
4. Generate launchd plist or systemd unit.
5. Lint service definition.
6. Run preflight.
7. Install only if preflight passes.
8. Enable only if install succeeds.
9. Start only if enable succeeds.
10. Emit status.

If preflight fails, activation MUST stage the service but MUST NOT enable persistence unless explicitly requested.

## 11. Policy checks

CI and local doctor checks SHOULD flag these patterns:

- `/Library/LaunchAgents` with non-Apple root-owned user agents.
- `KeepAlive=true` without bounded restart policy.
- `/tmp/*` service logs.
- `gcloud.auth.docker-helper` dependency in noninteractive runtime.
- Direct Google Artifact Registry image in persistent runtime command.
- `podman run` in launchd/systemd without preflight wrapper.
- Opaque Nix store wrapper used as the only operator-visible command.
- Missing uninstall/quarantine command.
- Excessive launchd run count or systemd restart count.

## 12. Regression cases

The local-agent runtime test suite MUST cover:

- Podman machine missing.
- Podman machine stopped.
- Podman socket refused.
- Local image missing.
- Local image present.
- Remote image tag present.
- Host Docker config contains Google credential helper.
- Empty authfile bypasses ambient credentials.
- Launchd disabled override exists.
- Duplicate label exists.
- Old system-wide plist exists.
- Container enters `Stopping`.
- Wrapper uses GNU-only syntax on macOS.
- Log directory missing.
- Nix store target stale or garbage-collected.

## 13. Definition of done

A SourceOS local agent is compliant when:

- It installs through `sourceos-agent` or Nix local-agent module.
- It passes preflight.
- It has a declared scope.
- It has no unbounded restart loop.
- It logs outside `/tmp`.
- It has a noninteractive-safe auth mode.
- It has status, logs, doctor, quarantine, and uninstall commands.
- It can be explained to an operator without reading opaque Nix store paths.
- It fails closed when dependencies are unavailable.

## 14. Immediate migration target: node-commander

The first migration target is `node-commander`.

Required migration:

1. Remove legacy system-wide LaunchAgent behavior.
2. Generate a user-scoped service declaration.
3. Use local image tag: `localhost/socioprophet/node-commander:dev-boot`.
4. Use explicit empty authfile at runtime.
5. Add Podman machine and image preflight.
6. Write logs under `~/Library/Logs/SocioProphet` or `~/Library/Logs/SourceOS`.
7. Add status/doctor/logs/quarantine commands.
8. Resolve container lifecycle state transitions such as `Stopping`.
9. Replace heartbeat-only test image with the intended product behavior.
10. Add regression tests for the original failure chain.
